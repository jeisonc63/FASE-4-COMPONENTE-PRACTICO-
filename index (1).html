Módulo de Servicios

Tres tipos de servicios especializados

Cada servicio hereda de la clase abstracta Servicio e implementa calcular_costo() y describir() de manera polimórfica.

🏛️

Sala de Reunión

Reserva por horas con precio base de $50.000 COP/h. Suplemento automático para grupos de más de 5 personas. Validación de capacidad máxima.

calcular_costo(horas, descuento, impuesto)
💻

Alquiler de Equipos

Laptops ($25k), proyectores ($15k) y servidores ($80k) por unidad/hora. Soporta múltiples unidades simultáneas. Validación de tipo de equipo.

precio_hora × cantidad × horas
🎓

Asesoría Técnica

Tarifas por nivel del asesor: Junior ($80k), Senior ($150k), Expert ($250k). Especificación de área temática. Soporte para descuentos e IVA.

tarifa_nivel × horas
Arquitectura OOP

Cinco principios aplicados rigurosamente

El sistema demuestra cada pilar de la programación orientada a objetos con ejemplos concretos en el código.

Abstracción EntidadBase · Servicio
class Servicio(EntidadBase): @abstractmethod def calcular_costo(self, horas, descuento=0.0, impuesto=0.0): ... @abstractmethod def describir(self) -> str: ...
Herencia 3 subclases de Servicio
EntidadBase └── Servicio (ABC) ├── SalaReunion ├── AlquilerEquipo └── AsesoriaTecnica
Polimorfismo Mismo método, distintos resultados
for srv in [sala, equipo, asesoría]: print(srv.describir()) # → comportamiento único # por tipo de servicio
Encapsulación Atributos privados + propiedades
@email.setter def email(self, valor): if not PATRON.match(valor): raise ClienteInvalidoError( "email", valor, "formato inválido")
Sobrecarga Parámetros opcionales
sala.calcular_costo(2) sala.calcular_costo(2, descuento=0.10) sala.calcular_costo(2, impuesto=0.19) sala.calcular_costo(2, descuento=0.10, impuesto=0.19)
Excepciones try/except/else/finally + chain
try: reserva.confirmar() except ReservaInvalidaError as e: logger.error(str(e)) else: logger.info("Confirmada ✓") finally: logger.info("Intento procesado")

Manejo de Errores

Jerarquía de excepciones personalizadas

Todas heredan de SoftwareFJError. Cada error tiene código único, mensaje descriptivo y se registra automáticamente en el log.

1001
ClienteInvalidoError
Email malformado, nombre vacío o teléfono con formato incorrecto.
2001
ServicioNoDisponibleError
Servicio con parámetros incorrectos o fuera de operación.
2002
CapacidadExcedidaError
Número de asistentes supera el límite de la sala reservada.
2003
CostoInconsistenteError
Cálculo de precio produce resultado negativo, infinito o NaN.
3001
ReservaInvalidaError
Duración negativa, cambio de estado no permitido o servicio ya cancelado.
4001
ParametroFaltanteError
Argumento obligatorio enviado como None u omitido en la llamada.
